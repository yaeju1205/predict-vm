# Branch
분기에서 특정 분기가 실행될 확률을 예측합니다
분기가 실행될 확율을 예측하여 더 많이 사용하는 변수를 예측합니다

## 1. 2-bit Saturating Counter (비트 포화 카운터)
4개의 상태를 가진 유한 상태 기계 입니다
- 00 강하게 취하지않음
- 01 약하게 취하지않음
- 10 약하게 취함
- 11 강하게 취함

분기가 실제로 일어날때 비트를 더합니다

## 2. Correlated Branch Prediction (상관 분기 예측)
특정 분기 결과가 다른 분기와 관계가 있다는점을 이용합니다

## 3. 분기 목적지 버퍼 BTB (Branch Target Buffer)
일종의 캐시 메모리로 분기 명령어의 주소와 그 명령어가 이전에 실제로 점프했던 목적지 주소를 쌍으로 저장합니다

명령어 구성 단계에서 현재 pc 값이 BTB에 있는지 확인하고, 있다면 즉시 목적지 주소로 다음 명령어를 가져옵니다

## 4. Neural Branch Prediction (신경망 분기 예측)
아주 작은 신경망 유닛을 사용합니다
- y = y=∑ i=0 n wi * xi​
- xi 는 최근 분기 이력입니다 (1 또는 -1)
- y 가 0 보다 크다면 취함 0 이하라면 취하지 않음 으로 판단합니다
학습 과정은 `docs/Branch/Neural.md` 를 참고하세요

## 5. Misprediction Penalty (예측 실패 시의 페널티)
예측이 틀렸을 경우 다음과 같은 처리를 합니다
- 잘못된 예측으로 미리 실행 중이던 모든 명령어를 파기합니다
- 잘못된 연산 결과가 반영된 레지스터 상태를 분기 시점 이전으로 되돌립니다
- 올바른 주소에서 명령어를 다시 가져옵니다 (optional)

## 6. Branch-Predict Entropy (분기 예측 엔트로피)
분기들의 예측확률이 동등하다고 판단될 경우 분기예측이 아닌 분기들이 공통적으로 사용하는 변수를 레지스터에 올리려 안전하게 사용합니다
